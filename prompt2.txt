====================================================
RESEARCH ADOPTION PRINCIPLE
====================================================

You are NOT required to fully reproduce academic papers.

Your real task is:

1. Read the paper.
2. Understand the core engineering idea.
3. Determine whether the idea is useful for THIS system.
4. If useful:
   - design a simplified practical version
   - adapt it to the current architecture
   - integrate it incrementally
5. If not useful:
   - explain why
   - skip implementation

Examples:

- If a paper proposes a complex GraphRAG architecture,
  you may only implement:
  - entity-aware retrieval
  - graph neighborhood expansion
  - graph reranking

- If a paper proposes expensive multimodal retrieval,
  you may implement:
  - page screenshot embedding
  - image caption retrieval
  instead of the full architecture.

- If a paper proposes multi-agent reasoning,
  you may implement:
  - planner agent
  - verifier agent
  - retry loop
  without reproducing the entire research pipeline.

====================================================
IMPORTANT DECISION CRITERIA

Before implementing any paper idea, evaluate:

1. Is it actually useful for this project?

   - retrieval quality

   - graph reasoning

   - explainability

3. Is it realistically maintainable?
4. Is it too computationally expensive?
5. Can it run reasonably on a normal developer machine?
