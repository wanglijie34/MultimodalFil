# InsightGraph Agent - 论文技术集成评估

本文档评估了集成 `prompt2.txt` 中研究技术后的系统表现。

## 📊 集成技术对比

| 技术灵感 | 集成功能 | 解决的痛点 | 定性评估 |
| :--- | :--- | :--- | :--- |
| **RAPTOR** | 层级化摘要索引 (`is_summary`) | 传统 RAG 难以回答全局性、总结性问题。 | **显著提升**。对于“这份文档讲了什么”这类问题，系统现在能先检索页面摘要。 |
| **LightRAG** | 实体邻居上下文扩展 (`get_entity_context`) | 向量检索对具有关联关系的实体识别较差。 | **提升明显**。在检索到某个实体时，能带出其在图谱中的相关关系，使回答更全面。 |
| **Agentic RAG** | 多步执行轨迹展示与角色分工 | 问答过程黑盒化，用户难以验证。 | **提升透明度**。用户在前端能清晰看到 Router 和 Retrieval 的工作状态。 |

## 📈 性能与开销权衡

1.  **延迟 (Latency)**:
    *   **摘要生成**：增加了文档摄入时的延迟（约每个页面增加 2-5 秒 LLM 调用）。
    *   **检索速度**：混合检索和图搜索增加了约 200-500ms 的检索延迟，但在可接受范围内。
2.  **存储 (Storage)**:
    *   **Qdrant**：向量点位增加了约 10-20%（由于增加了摘要向量）。
    *   **PostgreSQL**：数据库表体积略微增加。
3.  **Token 消耗**:
    *   摄入阶段增加了总结成本。
    *   回答阶段由于 Context 包含了图谱信息和摘要信息，上下文长度增加了 20-30%。

## 🔍 示例对比 (定性)

**Query**: "比较文件 A 和文件 B 中关于技术 X 的定义。"

*   **基线版本 (Simple RAG)**: 仅检索到包含 "技术 X" 的几个片段，可能漏掉一个文档中的核心定义。
*   **改进版本 (Hierarchical + Graph)**: 首先检索摘要层定位到技术 X 的核心章节，同时通过图谱关联出技术 X 的相关组件。回答更具对比性和完整性。

## 🛠️ 后续建议

1.  **动态层级控制**：目前的层级是固定的（Level 1），对于特长文档，可以支持递归生成多级摘要。
2.  **视觉模态完全落地**：目前仅实现了数据模型，下一步需要真正接入 PDF 转图片的预览功能（ColPali 灵感）。
